On the Implementation of a Peer Production License

Background

  • Kleiner and Bauwens point to the need for transvestment, and propose a Peer Production License.
  • This would require sharing information selectively with 'blessed' individual or organizations (Peers). This seems prima facie counter to the notion that 'useful information wants to be free'.
  • But, Kleiner has elsewhere made the point: privacy is necessary in order to achieve true consent when sharing. If you don't have the ability to withold consent, then you can't truly give consent.
  • The same values and infrastructure that would uphold this perceived right to Privacy -- the ability to decide with whom one shares one's information -- seem, then, at least potentially consistent with the dynamics of sharing in a Peer Production Licensing context, in which information is shared only with Peers.
  • Selectively sharing useful information with particular Peers seems like it could be consistent with the same values and practices that support the maintenance of Privacy.

The DAT protocol: secure content addressing for file sharing

  • The DAT protocol provides a secure 'hash'-based content-addressing system for file storage.
  • Every file is assigned a unique identifier that is a function of its contents. This identifier is difficult to guess without knowing the contents of the file.
  • Any change or addition to the file results in a different hash address for the file.
  • Thus, every new 'version' of a file will have a new, difficult-to-guess, effectively-unique identifier.

Proposal: Peer Production Licensing, via DAT

  • A commons-oriented individual or organization (the 'Publisher') might decide to share a set of files with particular Peers, using the DAT protocol.
  • The licensing might be something like: "Free for education, non-commercial, and commons-oriented uses."
  • The definition of 'commons-oriented' would be at the discretion of the Publisher.
  • The Publisher uses the criterion of 'commons-oriented' to decide whether or not some entity (individual, organization) qualifies as a Peer.
  • Every new version of a file would necessarily have a new DAT address (because: content addressing).
  • When creating a new version of a file, the Publisher only shares the resultant new address with Peers.

Challenge: controlling the flow of information

It seems technically challenging, in an era in which the marginal cost of sharing information is approaching zero, to prevent the unauthorized sharing of information by Peers with Non-Peers. Once a data file is available to a Peer, what is to prevent that Peer from sharing information with Non-Peers -- using the DAT or other protocols?

Traceablity as a solution?

  • The Publisher might include an 'address' in a shared file that references a particular Peer -- adding to the file a secret key, only shared with that Peer.
  • Then, if any DAT file Published by the author is found in the hands of a Non-Peer, its origin could be traced to the Peer who shared it in an unauthorized way.
  • This still doesn't prevent unauthorized sharing of files via other, non-traceable means.
  • But at least this would indur an extra computational cost on the part of Peers who attempt to share information with Non-Peers.
